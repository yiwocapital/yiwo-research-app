# 新闻搜索

在历史市场新闻数据中搜索特定主题。

## 前置条件

本 skill 需要先安装 `yra-news` CLI 并完成认证。如果尚未安装，请先运行 `yra-setup` skill。

## 工作流程

### 步骤 1：解析搜索意图

从用户查询中提取：

| 参数 | 示例 | 默认值 |
|------|------|--------|
| 关键词 | "美联储"、"原油"、"美联储 加息"、"inflation" | （必填） |
| 日期范围 | "今天"、"昨天"、"本周"、"20260601-20260605" | 今天 |
| 重要性 | "只看很重要"、"所有级别" | 全部 |
| 来源 | "只看国内"、"国外来源" | 全部 |

**日期解析规则**：
- "今天" / "today" → 当前日期（YYYYMMDD）
- "昨天" / "yesterday" → 前一天
- "本周" / "this week" → 本周（周一到今天）
- "N天前" → 今天 - N 天
- "YYYYMM" → 整月（使用日报文件）
- "YYYYMMDD" → 指定日期（使用小时文件）

### 步骤 2：验证 `yra-news`

与 `yra-news-summarize-today` 相同：检查安装和认证。

### 步骤 3：确定数据源

根据日期范围：

| 范围类型 | 拉取文件 | 命令 |
|----------|----------|------|
| 单日 | 小时文件 | `yra-news list-hours --date YYYYMMDD` |
| 多日 | 日报文件 | `yra-news list-dailies --month YYYYMM` |
| 整月 | 日报文件 | `yra-news list-dailies --month YYYYMM` |

### 步骤 4：拉取并解析数据

小时搜索：
```bash
yra-news list-hours --date YYYYMMDD --format json
# 对每个文件：
yra-news get-hour --file XXX --format json
```

日报搜索：
```bash
yra-news list-dailies --month YYYYMM --format json
# 对范围内每个文件：
yra-news get-daily --file XXX --format json
```

### 步骤 5：过滤消息

按以下顺序应用过滤：

1. **关键词匹配**：在 `message.content` 中搜索（不区分大小写，支持中文）
2. **重要性过滤**：如果指定，则匹配 `message.importance`
3. **来源过滤**：如果指定，则匹配 `message.source`
4. **日期过滤**：已通过文件选择应用

**关键词匹配策略**：
- 多词查询（如"美联储 加息"）：匹配所有词（AND 逻辑）
- 单个词：精确匹配或子串匹配
- 英文关键词不区分大小写

### 步骤 6：格式化结果

```markdown
# 🔍 搜索结果: "[关键词]"

**搜索条件**: [日期范围] | [重要性过滤] | [来源过滤]
**匹配消息**: N 条

---

## 结果列表

### 1. [来源] YYYY-MM-DD HH:MM
**重要性**: [🔴/🟡/⚪]

[消息内容全文]

---

### 2. ...
```

### 步骤 7：提供上下文

结果之后添加：

```markdown
## 📊 统计

| 维度 | 分布 |
|------|------|
| 按日期 | 2026-06-05: N 条, 2026-06-04: N 条 |
| 按来源 | 国内: N, 国外: N, 彭博社: N, 路透社: N |
| 按重要性 | 很重要: N, 重要: N, 普通: N |
```

## 高级搜索模式

### 时间相关查询
- "昨天关于原油的消息" → 昨天，关键词="原油"
- "本周美联储有什么动作" → 本周，关键词="美联储"
- "6月1日到5日通胀相关" → 日期=20260601-20260605，关键词="通胀"

### 复合查询
- "昨天很重要 + 国内" → 日期=昨天，重要性=critical，来源=国内
- "本周国外关于科技的消息" → 本周，来源=国外，关键词="科技"

### 无结果处理

如果未找到匹配项：
```markdown
未找到匹配 "[关键词]" 的消息。

**建议**:
- 尝试更宽泛的关键词
- 扩大日期范围
- 检查是否有拼写错误
```
